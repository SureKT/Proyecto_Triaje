El problema

272 entrevistas sin etiquetar

  • Transcripciones de entrevistas médico-paciente en texto libre
  • Ninguna tiene etiqueta de urgencia asignada por un médico
  • El Sistema de Triaje Manchester exige clasificar en C1–C5 en segundos
  • Objetivo: automatizar ese triaje a partir de la transcripción

Solución: LLM actúa como anotador clínico → genera etiquetas estructuradas → Random Forest entrena sobre esas columnas → sistema desplegable en tiempo real

272 .txt→ LLM etiqueta→ RF aprende→ Inferencia en vivo

Fase 1 · Orquestación LLM

El LLM como anotador clínico

Técnicas de prompting
  • Role prompting — "eres experto en triaje MTS"
  • Few-shot — 2 casos calibrados (SCA real vs pánico)
  • Structured Outputs — format=json, JSON obligatorio
  • temperature=0 — reproducibilidad total
  • Regla de oro — ante duda, nivel más urgente
LLM elegido

Batch 272 casos — Ollama llama3.1:8b

Sin rate limits · sin datos a terceros · GPU local · determinista

Demo portátil — OpenRouter Gemini Flash

Sin GPU requerida · ~3-5 s/pred · modelo gratuito · misma API

Fase 1 · Ground Truth

¿Es válido el LLM como ground truth?

⚠️ Limitación conocida: las etiquetas las genera el LLM, no un médico real. El RF aprende de esas etiquetas.

Riesgo
  • LLM comete errores sistemáticos en casos límite C2/C3
  • RF hereda ese sesgo durante el entrenamiento
Mitigación
  • temperature=0 → errores deterministas y detectables
  • Few-shot calibrado → ancla criterio Manchester
  • Regla de oro → reduce under-triage sistemático

En producción real: validación clínica de una muestra de 30-50 etiquetas por médico de urgencias sería el siguiente paso.

Fase 2 · Normalización semántica

De texto libre a vector numérico (ml_features.py)

Texto libre → número
  • "me cuesta respirar" → disnea: 1
  • "dolor muy fuerte" → dolor_intensidad: 8
  • No mencionado → -1 (no 0, no media)
  • "M" / "F" / null → 1 / 0 / -1

-1 ≠ 0: dolor=0 → sin dolor. dolor=-1 → no consta en la transcripción.

Fase 2 · Técnica de balanceo

class_weight='balanced' vs SMOTE

SMOTE — descartado
  • Genera C2 sintéticos interpolando entre 9 reales
  • Features mixtas (bool + int + -1) → SMOTE-NC necesario, más complejo
  • Resultado: datos clínicos inventados, poco realistas
class_weight='balanced' — elegido
  • No toca los datos — solo repondera el criterio de error
  • C2: peso ~15×, C3: peso ~0.68× (inversamente proporcional)
  • Recall C2: 0.0 → 0.970 tras añadir este parámetro
# Pesos resultantes (aprox.):
{1: 45.0,  2: 15.0,  3: 0.68,  4: 3.1,  5: 5.0}   # C2 vale 15 veces más que un error en C3

¿Por qué 0.850 y no 0.969?
C1 tiene soporte=0 en el dataset — F1=0.0 arrastra la media. Pacientes C1 no generan entrevistas (llegan inconscientes). Ausencia correcta del dominio.

¿Por qué F1 macro y no accuracy?
Con 73% de C3, predecir siempre C3 da accuracy=73% sin aprender nada. F1 macro penaliza igual todos los niveles sin ponderar por frecuencia.

Fase 3 · Auditoría clínico-ética

Detección de under-triage por sesgo emocional

El problema clínico
  • Paciente muy ansioso describe pánico, no síntomas objetivos
  • LLM puede asignar C2; RF puede predecir C3
  • ¿Quién tiene razón? → caso para revisión médica
-- Criterio de under-triage (/metricas/auditoria):
WHERE prediccion_modelo > nivel_triaje
  AND score_ansiedad >= 0.7
Doble rol de score_ansiedad

Feature del RF (2ª más importante, 13.9%)

El RF aprende que ansiedad alta ≠ urgencia alta → corrige sesgo emocional del LLM

Señal de auditoría ética

Si RF baja el nivel Y ansiedad > 0.7 → caso marcado para revisión médica obligatoria

No acusa al modelo: informa al médico que hay discrepancia que merece revisión

Fase 3 · Valoración automática

Fórmula de valoración (0-10)

valoración = max(0, confianza_RF − |pred_RF − nivel_LLM| × 0.25) × 10

Limitaciones y mejoras futuras

Limitaciones conocidas
  • Ground truth LLM-generado, no validado por médico real
  • 272 muestras — pequeño para producción clínica real
  • Entrenado en inglés · demo en español (no evaluado específicamente)
  • Crisis de pánico con diagnóstico previo confirmado → posible sobre-triaje
Mejoras futuras
  • antecedente_ansiedad como feature booleana adicional
  • 3.er ejemplo few-shot (pánico + diagnóstico confirmado)
  • Validación clínica de 30-50 etiquetas por médico de urgencias
  • SHAP/LIME — explicabilidad por caso individual (no solo global)
